# First Stockpile Walkthrough — MetaRDU Industrial

> **5-minute guide**: Process your first stockpile volume calculation
> from LAS file to signed audit report.

## Prerequisites

- MetaRDU Industrial installed (Tauri build)
- A LAS file of a stockpile survey (drone photogrammetry or TLS scan)
- No prior configuration needed — the app auto-detects everything

---

## Step 1: Launch the App

Double-click the MetaRDU Industrial icon. You'll see:

1. **Splash screen** (~2s) — animated theodolite reticle
2. **Module loading** (~3s) — geodesy, formats, mining modules initialize
3. **Workspace** — OpenLayers map canvas with the sidebar

If this is your first run, you'll see the **onboarding screen** first:
- Select your domain: **Mining** (for stockpiles)
- Pick your CRS: **EPSG:32756** (or your local UTM zone)
- Click **Continue**

---

## Step 2: Load Your LAS File

**Drag and drop** your `.las` file onto the map canvas, OR:

1. Click **+ New Survey** in the sidebar
2. Select your LAS file
3. The app reads the header and shows the file's bounds on the map
4. Wait for the green checkmark (✓) next to the filename — that means
   the probe is complete

> **Tip**: If you see "Terrain detected — not a stockpile", your file
> spans more than 500m. The EOM Auditor is designed for stockpile-sized
> areas. Crop your point cloud first, or use Volume Calc with a design
> surface for terrain.

---

## Step 3: Open the EOM Auditor

1. Press **Ctrl+K** to open the command palette
2. Type "eom" and select **EOM Volumetric Auditor**
3. OR: click the **EOM Auditor** item in the sidebar (Mining section)

The EOM Auditor dialog opens with:
- Current LAS path (auto-filled from your loaded file)
- Reference elevation (auto-detect ON by default)
- Customer, Site, Surveyor fields (fill these in for the audit report)

---

## Step 4: Configure the Pipeline

### Reference Elevation
- **✓ RANSAC auto-detect** (recommended) — the app finds the ground
  elevation automatically using histogram-mode detection. This gives
  ~1.4% error vs the true ground.
- **Manual** (uncheck the box) — type the known ground elevation if
  you have a surveyed bench mark. Gives ~1.5% error.

### Cell Size
- **1.0m** (default) — good for drone surveys at 0.5m point spacing
- **0.5m** — for high-density TLS scans
- **2.0m** — for large stockpiles (>100m diameter)

### Bench Interval
- **5.0m** (default) — standard bench height for most mines
- Set to **0** to disable bench breakdown (just total volume)

---

## Step 5: Run the Pipeline

Click the **Run Pipeline** button. You'll see progress through:

1. **Hashing** — SHA-256 of the source LAS (audit trail)
2. **Reading** — loading all points into memory
3. **Classifying** — CSF cloth simulation separates ground from stockpile
4. **Rasterizing** — IDW interpolation creates a 1m DEM grid
5. **Computing** — cut/fill volumes against the reference plane
6. **Auditing** — sealing the audit hash + chain of custody

**Runtime**: ~0.15s for 10K points, ~2s for 100K points.

---

## Step 6: Review Results

The results panel shows:

```
┌─────────────────────────────────────────┐
│ Fill volume:       3,397.88 m³          │
│ Cut volume:           10.21 m³          │
│ Net volume:        3,387.67 m³          │
└─────────────────────────────────────────┘

Fill cells: 1,967  Cut cells: 634  Cell area: 1.0 m²
```

- **Fill** = material above the reference plane (the stockpile)
- **Cut** = material below the reference plane (excavation)
- **Net** = Fill - Cut (positive = net fill, negative = net cut)

The **bench breakdown** table shows volume per elevation band —
useful for per-bench production reporting.

The **audit hash** (e.g., `6669509137a7b774`) is a SHA-256 fingerprint
of the entire pipeline run. Anyone with `metardu-verify` can confirm
the report was generated from this exact LAS file with these exact
parameters.

---

## Step 7: Generate the PDF Report

1. Fill in **Customer**, **Site/Pit**, and **Surveyor** fields
2. Set the **PDF report output** path (or click **Save As** to browse)
3. Click **Generate Report**

The PDF includes:
- Volume summary (fill/cut/net)
- Bench breakdown table
- Chain-of-custody record (custodian, timestamp, audit hash)
- Pipeline parameters (CSF settings, cell size, bench interval)
- Signature block (if running under a paid license)

> **Trial mode**: 3 reports free. After that, a license is required
> for signed reports. Unsigned reports can still be generated for
> evaluation.

---

## Step 8: Verify the Report (Optional)

```bash
metardu-verify /path/to/eom_report.pdf
```

The standalone verifier confirms:
- The PDF's audit hash matches the source LAS file
- The chain-of-custody record is intact
- The report was generated by MetaRDU Industrial

This is the trust anchor for commercial reconciliation — the
surveyor can prove the report wasn't tampered with after generation.

## Accuracy Expectations

| Scenario | Expected Error | Why |
|----------|---------------|-----|
| Synthetic cone (known volume) | ~1.4% | IDW interpolation + GPS noise |
| Real drone stockpile | ~2-5% | Depends on point density + GPS accuracy |
| Real TLS stockpile | ~1-3% | Higher density → better interpolation |
| Against a design surface (DXF TIN) | ~1-2% | Direct cell-by-cell comparison |

**If your error seems too high:**
- Check that the point cloud is dense enough (>1 pt/m²)
- Check that the reference elevation is correct (try manual mode)
- Check that the cell size matches the point spacing
- Make sure the file is a stockpile, not terrain

---

## Troubleshooting

### "Empty point cloud" error
The LAS file has `point_count = 0` in the header, or the file is
corrupted. Try re-exporting from your photogrammetry software.

### Pipeline hangs
The point cloud is too large or too sparse. The CSF filter auto-tunes
for density, but extremely large files (>50M points) may need to be
tiled first. Use `lastile` or CloudCompare to split the file.

### "Terrain detected" warning
The file spans >500m — likely airborne lidar, not a stockpile survey.
Crop to the stockpile area, or use Volume Calc with a design surface.

### Volumes seem wrong
1. Check the auto-detected reference elevation (uncheck auto, try
   manual with a known bench mark)
2. Check that CSF classified enough points as ground (should be >50%
   for a stockpile on flat ground)
3. Check the DEM grid size — if it says "auto-scaled", the cell size
   was increased to fit a large extent
